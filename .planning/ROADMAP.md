### Phase 1: Enhanced Metric Insights
**Goal**: Users get detailed, actionable explanations for every discovered metric
**Depends on**: Nothing (builds on existing v1 infrastructure)
**Requirements**: INSI-01, INSI-02, INSI-03
**Success Criteria** (what must be TRUE):
  1. Clicking a metric shows a detailed business explanation generated by the LLM
  2. Analysis stage logs show LLM reasoning steps, not just "processing..."
  3. Each metric has a categorization with business impact score

### Phase 2: Pipeline Robustness
**Goal**: Analysis pipeline handles all LLM failure modes gracefully
**Depends on**: Phase 1
**Requirements**: RBST-01, RBST-02, RBST-03
**Success Criteria** (what must be TRUE):
  1. Malformed JSON from LLM is recovered without losing data
  2. Truncated LLM outputs are detected and re-requested
  3. Provider failures trigger retry with exponential backoff
**Plans**: TBD

Plans:
- [ ] 02-01: Robust JSON parser with partial recovery

- [ ] 02-03: Provider-level retry with exponential backoff
